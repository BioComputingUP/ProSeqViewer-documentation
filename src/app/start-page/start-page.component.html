ProSeqViewer

ProSeqViewer library for the visualization of (not only) biological sequence features. Used in MobiDB, DisProt and RepeatsDB.
Based on Typescript and compatible with Angular 2+ framework. A JavaScript version of the package is also available (see the API documentation).

Indexes Location

Indexes are disabled by default, you can activate them and choose indexes location.

Lateral indexes

Sequence indexing is placed at the beginning of each line. It is also possible to set a value for the startIndex (set to 50 in this example). 

{{latindexoptions | json}}

Indexes on top

Sequence indexing is right aligned on top of each block of amino acids.

{{topindexoptions | json}}
Wrap Line

Each letter is encapsulated in a column element (just one cell for single sequences) which is part of a block of columns. Blocks are graphically separated to improve readability and can have different length (setting chunkSize option).

Responsive (default)

By default, blocks are automatically moved to a new line when they exceed the container width (responsive behavior). 

{{wrapoptionsnr2 | json}}

Non-responsive

Block wrapping can be disabled if the user wants to slide MSAs horizontally (non-responsive behavior).

{{wrapoptionsnr1 | json}}
Icons
Add icons on the top of the sequences to indicate the secondary structure.You can use the icons already available (helix, turn, lollipop, strand, arrowLeft, arrowRight, noSecondary) or you can input custom svg icons (check the Documentation.)
This example shows the secondary structure of the ASL protein mapped onto the secondary structure of the human ASL protein (P04424, Homo sapiens) sequence. Helices are shown in pink and beta strands are shown as yellow arrows.

Sequence Color Matrix
Set a global sequence color matrix. Available global sequence color matrix is blosum62. Here amino acid pairs are compared, in particular each residue is compared with the residue at the same position in the first sequence. The aminoacid pair is colored based on a input matrix.

Default matrix

Blosum62 is a default matrix, but it is also possible to set a custom matrix (see Custom matrix).

{{blosumdefaultoptions | json}}

Custom matrix

It is possible to set a custom sequence color matrix, check the Documentation (Input -option, sequenceColorMatrix-) for details.

{{blosumcustomoptions | json}}
Consensus
A consensus sequence will be the generated on top of the alignment, representing the most frequent aminoacid (consensus identity) or property (consensus physical) found at each position in a sequence alignment.
As a default, the consensusDotThreshold is set at 90%, meaning that position with a frequency less of 90% will display a dot.
Consensus 'identity' and 'physical' can't be set at the same time. In case of overlapping settings, consensus 'identity' will be displayed.

It is also possible to set a custom consensus with personalized colors, levels and symbols and modify the consensusDotThreshold, check the Documentation for details.

Identity consensus

In the consensus identity the aminoacid found at each position is colored based on its frequency in the alignment. Each level has a color: (100%, 80%, 70%..)
